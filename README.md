# Incident Automation — Scoped App (Studio Source Control Export)

This repo is the **native ServiceNow Studio Source Control export** of the Incident Automation scoped application — pushed directly from ServiceNow Studio via its built-in Git integration, not manually exported.

📌 **For the full project writeup** — problem statement, architecture, the Business Rule script, notification configs, and the real debugging story — see the main portfolio repo:
👉 [servicenow-incident-automation](https://github.com/abhijeetpatil4157/servicenow-incident-automation)

## What's different about this repo

The main portfolio repo documents the *manual* Update Set XML export method. This repo represents a second, deliberate pass at the same project — rebuilt natively inside a custom scoped application (`Incident Automation`) and connected directly to GitHub through Studio's **Source Control** integration (repo URL + Personal Access Token), rather than exporting an Update Set by hand.

It demonstrates:
- Studio's **Commit changes** action (commit + push in a single step)
- An auto-created **per-instance branch** (`sn_instances/dev310914`)
- A real **Pull Request → merge** into `main`

## Contents

The `9c49425ac302cf1061b532a77d0131eb/` folder is the app's `sys_id`-named export directory — Studio's standard structure for a scoped app's tracked metadata (table definition, dictionary entries, the Business Rule, and both Email Notifications).

`sn_source_control.properties` is auto-generated by Studio when the app is first linked to a Git repository.
